## 2025年1月 — 项目启动与架构设计

### 1月6日 — 项目立项

今天正式确定了 Pool AI Knowledge 项目。核心目标：做一个基于 RAG 的智能知识库，让团队的文档和经验能被 AI 理解和检索。

技术选型讨论了一整天，最终确定：
- 后端用 **FastAPI**，异步性能好，自带 Swagger 文档
- 前端用 **Vue 3 + Element Plus**，开发效率高
- 数据库用 **MySQL**，成熟稳定
- 向量检索用 **FAISS**，内存级速度
- AI 模型用 **Google Gemini**，性价比高

### 1月13日 — 数据库设计完成

设计了四张核心表：
- `posts` — 文章/知识内容
- `admin_users` — 管理员账户
- `api_keys` — API 密钥管理
- `system_config` — 系统配置

用纯 SQL 文件管理 schema，所有建表语句都用 `CREATE TABLE IF NOT EXISTS`，保证幂等性。

### 1月20日 — 后端 API 框架搭建

完成了基础 API 框架：
- 统一响应格式 `R.ok(data)` / `R.fail(code, message)`
- JWT 认证（HS256，24小时过期）
- bcrypt 密码哈希
- 全局异常处理器

遇到一个坑：`bcrypt` 新版本和 `passlib` 不兼容，锁定 `bcrypt==4.0.1` 解决。

### 1月27日 — 管理后台 CRUD 完成

管理后台的文章增删改查、API 密钥管理、管理员管理全部完成。文章支持 Markdown 格式，标签用逗号分隔存储，API 返回时转为数组。

API 密钥在接口返回时做了遮蔽处理，只显示前4位和后4位，保证安全性。
